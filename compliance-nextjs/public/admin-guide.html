🔐2.1  Admin Login

Admin access requires credentials. The admin login page is separate from the user dashboard.

  1. Navigate to the Admin Login page
    Go to http://localhost:3000/admin/login — you will see the login form.
  2. Enter your admin username and password
    Type your credentials in the Username and Password fields. Default first-time credentials: username admin, password Admin@123.
  3. Click the Login button
    If credentials are correct, you are redirected to the Admin Dashboard. If incorrect, an error message appears below the form. Re-enter and try again.

📸 Admin Login page — enter username and password to access the admin panel

🔒 Security Note

Your admin session is stored securely in the browser. If you close the browser or the session expires, you will need to log in again. Always log out when using a shared computer.

📊2.2  Dashboard Overview

After logging in, you land on the Admin Dashboard which shows the main navigation and summary panels.

📸 Admin Dashboard — main navigation and summary

  1. The left sidebar shows navigation
    Items: User List (submissions), Check-In Table (overview grid), Notifications (deadline alerts), Export (Excel download).
  2. Summary numbers are shown on cards
    Quick counts show total submissions, pending reviews, approved, and rejected records.
  3. Use the sidebar to navigate between sections
    Click any sidebar link to go directly to that section. The current section is highlighted.
👥2.3  User List — View, Filter & Edit Submissions

The User List is the main management area. It shows all compliance submissions as a table with filtering, sorting, inline editing, and review tools.

📸 User List — all submissions shown as a filterable, editable table

Filtering and Searching
  1. Use the Project filter dropdown
    Click the Project multi-select dropdown at the top. Tick specific project names to show only those members, or leave all blank for all projects.
  2. Use the text search field
    Type any keyword in the Search box to filter rows by name, account, or type.
  3. Use the Status filter
    Click the Status dropdown to show only Pending, Approved, or Rejected submissions.
  4. Filter by period
    Use the Month/Year date picker to show only submissions for a specific period.

📸 User List with filters applied — filtered by project and status

Reviewing Submissions (Image + AI Details)
  1. Find the submission row you want to review
    Scroll or filter the table to locate the user.
  2. Click the 👁️ Review icon in the Actions column
    A Review Modal appears showing: the uploaded screenshot, AI validation result, confidence score, detected device type, and any failure reasons.
  3. Review the image and AI analysis
    Check whether the screenshot shows all required items. The AI result shows what it found (or did not find).
  4. Change the status to Approved or Rejected
    Inside the modal, use the Status dropdown to set Approved ✅ or Rejected ❌. Add a note in the Admin Note field if needed, then click Save.

📸 Review Modal — shows the full screenshot, AI validation details, and status controls

Inline Editing of Records

Any cell in the User List can be edited directly in the table — no separate edit screen is needed.

  1. Double-click (or click the ✏️ icon) on any editable cell
    The cell becomes an input field. Editable columns include: Name, Project, Team, Laptop Type, Status, Notes, and compliance tracking fields.
  2. Type the new value and press Enter (or click away)
    The change is saved automatically. A green flash confirms the save.
  3. To cancel an edit
    Press Escape while the cell is active — the original value is restored.

📸 Inline editing — click a cell to edit it directly in the table; press Enter to save

Adding / Removing Members
  1. To add a new member
    Click the + Add Member button above the table. Fill in the required fields (name, account, project, type) and click Save.
  2. To remove a member
    Click the 🗑️ Delete icon at the end of the row. A confirmation prompt appears — click Confirm to proceed.

⚠️ Deleting a member is permanent

Deleting a member removes their record and submission history. Consider setting status to Rejected or adding a note instead, to preserve the audit trail.

📁2.4  Tracking File Upload & Download

The system uses a tracking.xlsx Excel file as the master member list. Upload a new file to add or update members in bulk.

Upload tracking.xlsx
  1. Prepare your tracking.xlsx file
    The Excel file must have these columns: Name, Account, Project, Team, Laptop Type. Each row is one member. Save as .xlsx format.
  2. In the Admin Dashboard, go to User List
    Click the 📤 Upload Tracking button at the top of the User List page.
  3. Select your file
    Click Choose file, pick your tracking.xlsx, then click Upload. The system imports the data and merges it with existing submissions.
  4. Verify the data
    The table updates to show the imported members. Check that all rows and projects loaded correctly.

📸 Upload Tracking.xlsx button at the top of the User List page

Download Tracking
  1. Click the 📥 Download Tracking button
    The current tracking file is downloaded to your computer. Use it as a backup or to edit and re-upload.

✅2.5  Check-In Table

The Check-In Table provides a bird's-eye view of all users and their compliance status in a color-coded grid.

📸 Check-In Table — color-coded grid of all members and their compliance status

  1. Open the Check-In Table
    Click Check-In Table in the sidebar.
  2. Read the color codes
    Complete — Approved   Pending — Awaiting review   Missing — No submission
  3. Filter by Project
    Use the Project dropdown to show only a specific team's status.
  4. Filter by Status or Date Range
    Use the status and date-range filters at the top to narrow the grid.
  5. Click a cell to view the record
    Clicking a colored cell opens the review modal for that member's submission.

📊 Use for deadline tracking

Sort by the Missing filter and send a targeted reminder to only the members who have not yet submitted. Combine with the Notifications section to send a Teams message.

🔔2.6  Sending Notifications

The Notifications panel lets you send deadline reminders to all pending members via Microsoft Teams webhook.

📸 Notifications panel — configure message and send Teams reminder

  1. Open the Notifications section
    Click Notifications in the sidebar.
  2. Set the Deadline Date
    Click the Deadline Date field and pick the date by which all submissions must be done.
  3. Edit the notification message
    The default message is pre-filled. Click the text area to customise it — include the deadline date, what's required, and who to contact.
  4. Click Send Reminder
    Click the 📨 Send Reminder button. A confirmation message appears and the Last Sent timestamp updates.
  5. Verify delivery
    Check your Teams channel to confirm the message arrived. If it did not, check the webhook URL is correctly configured.

⏰ Automated reminders

The system can also be configured to send reminders automatically on a schedule (e.g. every weekday at 9 AM). Manual send is for ad-hoc announcements. Contact your system administrator to enable the automated schedule.

📸 Filled Notifications form ready to send — deadline date and custom message set

📥2.7  Export to Excel

Download the full compliance report as an Excel (.xlsx) file for offline analysis, sharing with management, or archiving.

  1. Navigate to the Admin Dashboard
    Any admin page works — the Export button is accessible from the top navigation bar or the sidebar.
  2. Click the 📥 Export to Excel button
    The file is generated and downloaded automatically by your browser. Check your Downloads folder.
  3. Open the downloaded file
    The file is named compliance-report-YYYYMMDD-HHmm.xlsx. Open it with Microsoft Excel or any compatible spreadsheet app.
  4. Review the contents
    The export includes all columns: Account, Name, Project, Type, Status, Submission Date, AI Confidence, Admin Notes, and compliance field data.

📸 Export button in the Admin Dashboard — click to download compliance-report.xlsx

ℹ️
What is included in the export:
  • All submission records (approved, pending, rejected)
  • AI validation result and confidence score
  • Admin review notes and status
  • Submission timestamps and device type
  • SEED compliance fields (for Windows/Mac types)

📅 Best practice — export at end of period

Export the report at the end of each compliance period (e.g. end of month). Archive each export file with the date in the filename so you have a historical record.
